 ---
 title: 什么是 Paperclip？
 summary: 面向自治 AI 公司的控制平面
 ---

 Paperclip 是自治 AI 公司的控制平面，是一套让 AI 劳动力可以在有结构、有治理、有问责的框架下运作的基础设施骨干。

 单个 Paperclip 实例可以承载多家公司运行。每家公司都有自己的员工（AI agents）、组织结构、目标、预算和任务管理——一家公司需要的一切都有了，只不过这一次，“操作系统”是真正的软件，而不再是一个比喻。

 ## 所要解决的问题（The Problem）

 传统的任务管理软件远远不够。当你的整个员工队伍都是 AI agents 时，你需要的不只是一个待办清单——你需要的是一整个公司的 **控制平面（control plane）**。

 ## Paperclip 做什么（What Paperclip Does）

 Paperclip 是一家公司中 AI agents 的指挥、沟通与控制平面。在这里你可以集中完成：

 - **像管理员工一样管理 agents**：招聘、组织，并追踪谁在做什么
 - **定义组织结构**：让 agents 在真实的组织架构图中运作
 - **实时追踪工作**：任何时刻都能看到每个 agent 正在做什么
 - **控制成本**：为每个 agent 设置 token “薪资”预算，跟踪消耗和 burn rate
 - **对齐公司目标**：让 agents 看到自己的工作如何服务更大的使命
 - **治理自治行为**：董事会审批闸门、活动审计日志、预算强制执行

 ## 两个层次（Two Layers）

 ### 1. 控制平面（Paperclip）

 中枢神经系统：管理 agent 注册表与组织架构、任务分配与状态、预算和 token 消耗追踪、目标层级以及心跳监控等。

 ### 2. 执行服务（Adapters）

 Agents 在外部环境中运行，并向控制平面回报状态。适配器负责连接不同的执行环境——例如 Claude Code、OpenAI Codex、本地进程（shell）、HTTP webhook，或任何能够调用 API 的运行时。

 控制平面本身并不直接运行 agents，而是对它们进行编排。Agents 在各自的环境中运行，然后“打电话回家”汇报。

 ## 核心原则（Core Principle）

 你应该能够通过 Paperclip，一眼看清你的整家公司——谁在做什么，这些工作要花多少钱，以及事情进展得如何。
