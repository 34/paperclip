# Paperclip UI 规范（Paperclip UI Spec）

状态：草稿  
日期：2026-02-17

## 1. 设计理念（Design Philosophy）

Paperclip 的 UI 是专业级控制平面，而非玩具仪表盘。应像全天在用的工具：快速、键盘优先、信息密度高但不杂乱，默认深色主题。每个像素都要有存在理由。

设计原则：

- **紧凑但可扫读。** 在少点击的前提下展示更多信息，用留白分隔而非填充。
- **键盘优先。** 全局快捷键：搜索（Cmd+K）、新建 issue（C）、导航。高级用户应很少用鼠标。
- **上下文内操作，少用弹窗。** 行内编辑优于对话框，下拉优于整页跳转，不无故打断用户心智上下文。
- **默认深色主题。** 中性灰而非纯黑，强调色仅用于状态与优先级，文字为主视觉。

### 色彩系统（Color System）

- 背景、表面/卡片、边框、主/次文字、强调色（交互）等使用 HSL 定义。状态色：Backlog/Todo/In Progress/In Review/Done/Cancelled/Blocked 均有对应色。优先级：Critical（红实心）、High（橙半实心）、Medium（黄描边）、Low（灰虚线描边）。

### 排版（Typography）

- 字体：系统字体栈（若加载则 Inter，否则 -apple-system 等）。正文 13px/1.5；标签/元数据 11px/大写字间距；标题 14–18px/半粗，不全大写。

### 图标（Icons）

全程使用 `lucide-react`。侧栏、状态、操作按钮均配图标；导航 16px，行内 14px。

## 2. 应用外壳（Application Shell）

三区布局：侧栏 + 面包屑栏 + 主内容区。（其余布局、路由、组件与 API 等见英文原文。）
