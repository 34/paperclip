# Agent 运行子系统规范（Agent Runs Subsystem Spec）

状态：草稿  
日期：2026-02-17  
受众：产品与工程  
范围：Agent 执行运行时、适配器协议、唤醒编排与实时状态投递

## 1. 文档角色（Document Role）

本规范定义 Paperclip 在保持运行时无关的前提下如何实际运行 agents。

- `doc/SPEC-implementation.md` 为 V1 基线契约。
- 本文档补充 agent 执行相关的具体子系统细节，包括本地 CLI 适配器、运行时状态持久化、唤醒调度与浏览器实时更新。
- 若与当前代码中的运行时行为冲突，以本文档为目标行为。

## 2. 已采纳意图（Captured Intent）

以下意图在本文档中明确保留：

1. Paperclip 与适配器解耦，关键在协议而非具体运行时。
2. 仍需默认内置适配器以便系统开箱可用。
3. 前两个内置为 `claude-local` 与 `codex-local`。
4. 这些适配器在宿主机上直接运行本地 CLI，非沙箱。
5. Agent 配置包含工作目录与初始/默认 prompt。
6. 心跳运行已配置的适配器进程，Paperclip 管理生命周期，退出时解析 JSON 输出并更新状态。
7. Session ID 与 token 用量必须持久化以便后续心跳恢复。
8. 适配器应支持状态更新（短消息 + 颜色）及可选流式日志。
9. UI 应支持 prompt 模板“药片”变量插入。
10. CLI 错误应在 UI 中完整（或尽量完整）可见。
11. 状态变更须通过服务端推送在任务与 agent 视图中实时更新。
12. 唤醒触发应由心跳/唤醒服务统一处理，至少包括：定时器间隔、任务分配时唤醒、显式 ping/请求。

## 3. 目标与非目标（Goals and Non-Goals）

### 3.1 目标

1. 定义支持多种运行时的稳定适配器协议。
2. 交付可用于生产的 Claude CLI 与 Codex CLI 本地适配器。
3. 持久化适配器运行时状态（session ID、token/成本用量、最近错误）。
4. 在一个服务中集中唤醒决策与排队。
5. 向浏览器提供实时的 run/任务/agent 更新。
6. 支持按部署配置的完整日志存储，避免 Postgres 膨胀。
7. 保持公司作用域与现有治理不变式。

### 3.2 非目标（本阶段）

1. 跨多主机的分布式执行 worker。
2. 第三方适配器市场/插件 SDK。
3. 对不输出成本的提供方做完美成本核算。
4. 超出基础保留期的长期日志归档策略。
