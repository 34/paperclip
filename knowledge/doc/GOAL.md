 # Paperclip

 **Paperclip 是自治经济的基础设施。** 我们正在搭建一套让自治 AI 公司得以运行的基础设施。我们的目标，是让基于 Paperclip 的公司总体创造的经济产出，可以对标世界上最大国家的 GDP。我们做的每一个决策，都应该服务于这一点：让自治公司更强大、更可控、更可扩展，而且真正落地。

 ## 愿景（The Vision）

 自治公司——由 AI 劳动力组成、具备真实组织结构、治理机制和问责体系的公司——将成为全球经济中的一股重要力量。不是一家公司，而是成千上万、数以百万计的公司。会出现一个全新的经济层，运行在 AI 劳动力之上，并通过 Paperclip 进行协调。

 Paperclip 本身不是那家公司，Paperclip 是让这些公司成为可能的东西。我们是控制平面、是神经系统、是运行层。每一家自治公司都需要结构化的组织、任务管理、成本控制、目标对齐以及人类治理。这就是我们在做的事情。对于自治公司而言，我们就像传统意义上“企业操作系统”之于人类公司——只不过这一次，这个操作系统是真正的软件，而不只是一个比喻。

 衡量我们成功与否的标准，并不是某一家公司是否跑得通，而是：Paperclip 是否成为自治公司构建时的默认基础；以及这些公司整体上，是否真正成为可以与国家级产出相抗衡的经济力量。

 ## 问题（The Problem）

 现有的任务管理软件远远不够用。当你的整个员工队伍都是 AI agent 时，你需要的不只是一个待办清单——你需要的是一整个公司的 **控制平面（control plane）**。

 ## 这是什么（What This Is）

 Paperclip 是一家公司中 AI agents 的指挥、沟通与控制平面。它是你集中完成以下事情的唯一入口：

 - **像管理员工一样管理 agent**：招聘、组织，并追踪谁在做什么
 - **定义组织结构**：让 agent 也在真实的组织架构图中运作
 - **实时追踪工作**：任何时刻都能看到每个 agent 正在做什么
 - **控制成本**：为每个 agent 设置 token“薪资”预算，跟踪消耗和 burn rate
 - **对齐公司目标**：让 agents 看到自己的工作如何服务更大的使命
 - **沉淀公司知识**：为整个组织提供共享的大脑

 ## 架构（Architecture）

 分为两层：

 ### 1. 控制平面（本软件）

 中枢神经系统，用来管理：

 - Agent 注册表与组织架构
 - 任务分配与状态
 - 预算与 token 消耗追踪
 - 公司知识库
 - 目标层级（公司 → 团队 → agent → 任务）
 - 心跳监控——知道哪些 agent 在线、空闲或卡住

 ### 2. 执行服务（适配器）

 Agents 在外部运行，并向控制平面回报状态。一个 agent 本质上就是一段被启动后会去完成工作的 Python 代码。适配器将不同的执行环境接入：

 - **OpenClaw** —— 初始目标适配器
 - **心跳循环（heartbeat loop）** —— 简单的自定义 Python 循环，持续检查、上报并执行工作
 - **其他环境** —— 任何能调用 API 的运行时

 控制平面本身并不直接运行 agents，它负责编排。Agents 在各自的环境中运行，然后“打电话回家”汇报状态。

 ## 核心原则（Core Principle）

 你应该能够通过 Paperclip，一眼看清你的整家公司——谁在做什么，这些工作要花多少钱，以及事情进展得如何。
