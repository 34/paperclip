# Paperclip 二次开发指南

## 开发环境搭建

### 1. 前置条件

```
Node.js >= 20
pnpm = 9.15.4
Git
(可选) PostgreSQL - 生产级开发
```

### 2. 开发流程

```bash
# 安装依赖
pnpm install

# 启动开发环境
pnpm dev
# → 自动启动: PGlite + 数据库迁移 + API Server + Vite UI

# 另开终端，运行测试
pnpm test

# 类型检查
pnpm typecheck

# 运行 E2E 测试
pnpm test:e2e:headed
```

### 3. 开发脚本

```bash
pnpm dev              # API + UI 热重载
pnpm dev:server       # 只启动 API
pnpm dev:ui           # 只启动 UI
pnpm dev:list         # 列出运行中的服务
pnpm dev:stop         # 停止服务
pnpm build            # 构建所有包
pnpm db:generate      # 生成数据库迁移
pnpm db:migrate       # 运行迁移
```

## 代码修改指南

### 修改数据模型

**步骤**：

1. 在 `packages/db/src/schema/` 修改或新增表定义
2. 在 `packages/shared/src/types/` 添加对应类型
3. 在 `packages/shared/src/validators/` 添加 Zod 验证器
4. 在 `packages/shared/src/constants.ts` 添加新常量/枚举
5. 运行 `pnpm db:generate` 生成迁移
6. 运行 `pnpm db:migrate` 应用迁移

**示例：添加新表**：

```typescript
// 1. packages/db/src/schema/my-table.ts
import { pgTable, uuid, text, timestamp, jsonb, integer } from 'drizzle-orm/pg-core'

export const myTable = pgTable('my_table', {
  id: uuid('id').defaultRandom().primaryKey(),
  companyId: uuid('company_id').notNull().references(() => companies.id),
  name: text('name').notNull(),
  status: text('status').notNull().default('active'),
  data: jsonb('data'),
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  updatedAt: timestamp('updated_at', { withTimezone: true }).defaultNow().notNull(),
})

// 2. packages/db/src/schema/index.ts - 添加导出
export * from './my-table'
```

```typescript
// 3. packages/shared/src/types/my-type.ts
export interface MyEntity {
  id: string
  companyId: string
  name: string
  status: 'active' | 'inactive'
  data?: unknown
  createdAt: string
  updatedAt: string
}
```

```typescript
// 4. packages/shared/src/validators/my-type.ts
import { z } from 'zod'

export const myEntitySchema = z.object({
  name: z.string().min(1),
  status: z.enum(['active', 'inactive']).default('active'),
  data: z.any().optional(),
})
```

### 添加 API 端点

**步骤**：

1. 在 `server/src/routes/` 添加路由文件
2. 在 `server/src/services/` 添加服务文件（业务逻辑）
3. 在 `server/src/app.ts` 挂载路由
4. 在 `ui/src/api/` 添加 API 客户端
5. 在 UI 页面中调用

**示例**：

```typescript
// server/src/routes/my-route.ts
import { Router } from 'express'
import { db } from '@paperclipai/db'

const router = Router()

router.get('/api/my-entity', async (req, res) => {
  const { companyId } = req.query
  const results = await db.select().from(myTable)
    .where(eq(myTable.companyId, companyId as string))
  res.json(results)
})

router.post('/api/my-entity', async (req, res) => {
  const { companyId } = req.query
  const data = req.body
  const [created] = await db.insert(myTable)
    .values({ ...data, companyId: companyId as string })
    .returning()
  res.status(201).json(created)
})

export default router
```

```typescript
// server/src/app.ts - 添加路由
import myRoute from './routes/my-route.js'
// ...
app.use(myRoute)
```

```typescript
// ui/src/api/my-entity.ts
import { api } from './client'

export const myEntityApi = {
  list: (companyId: string) =>
    api.get(`/api/my-entity?companyId=${companyId}`),
  create: (companyId: string, data: any) =>
    api.post(`/api/my-entity?companyId=${companyId}`, data),
}
```

### 添加 UI 页面

**步骤**：

1. 在 `ui/src/pages/` 创建页面组件
2. 在 `ui/src/App.tsx` 的 `boardRoutes()` 中添加路由
3. 如需侧边栏入口，修改 `ui/src/components/Sidebar.tsx`

**示例**：

```typescript
// ui/src/pages/MyEntity.tsx
import { useQuery } from '@tanstack/react-query'
import { myEntityApi } from '../api/my-entity'
import { useCompany } from '../context/CompanyContext'

export function MyEntityPage() {
  const { company } = useCompany()

  const { data: entities, isLoading } = useQuery({
    queryKey: ['my-entity', company?.id],
    queryFn: () => myEntityApi.list(company!.id),
    enabled: !!company?.id,
  })

  if (isLoading) return <div>Loading...</div>

  return (
    <div className="p-6">
      <h1 className="text-2xl font-bold">My Entities</h1>
      {/* 渲染列表 */}
    </div>
  )
}
```

```typescript
// ui/src/App.tsx - 在 boardRoutes() 中添加
<Route path="my-entity" element={<MyEntityPage />} />
```

## 二次开发场景分析

### 场景 1：自定义 Agent 角色

```
需求: 添加新的 Agent 角色类型（如 "analyst"）

修改文件:
  1. packages/shared/src/constants.ts - 添加角色常量
  2. packages/db/src/schema/agents.ts - (drizzle text 类型，无需改)
  3. packages/shared/src/types/agent.ts - 更新类型
  4. packages/shared/src/validators/agent.ts - 更新验证器
  5. ui/src/components/AgentConfigForm.tsx - 添加角色选项
  6. server/src/services/agent-instructions.ts - 可能需要角色特定逻辑
```

### 场景 2：自定义审批流程

```
需求: 添加新的审批类型

修改文件:
  1. packages/shared/src/constants.ts - 添加审批类型
  2. packages/db/src/schema/approvals.ts - (jsonb payload，无需改)
  3. server/src/services/approvals.ts - 添加新类型处理
  4. server/src/routes/approvals.ts - 添加新端点
  5. ui/src/pages/ApprovalDetail.tsx - 添加新类型 UI
  6. ui/src/api/approvals.ts - 添加新 API 调用
```

### 场景 3：自定义成本模型

```
需求: 添加新的计费维度

修改文件:
  1. packages/db/src/schema/cost_events.ts - 添加字段
  2. packages/shared/src/types/cost.ts - 更新类型
  3. server/src/services/costs.ts - 更新计算逻辑
  4. server/src/services/budgets.ts - 更新预算检查
  5. ui/src/pages/Costs.tsx - 更新成本展示
```

### 场景 4：自定义工作区策略

```
需求: 支持新的工作区隔离策略

修改文件:
  1. packages/db/src/schema/execution_workspaces.ts - 添加策略类型
  2. server/src/services/execution-workspaces.ts - 添加策略逻辑
  3. packages/adapter-utils/src/types.ts - 添加策略类型
  4. server/src/services/heartbeat.ts - 更新工作区解析
```

### 场景 5：外部系统集成（推荐用插件）

```
推荐方式: 创建插件，而非修改核心代码

优势:
  - 独立开发/部署
  - 不影响核心系统稳定性
  - 可与其他用户共享

参考: packages/plugins/examples/plugin-kitchen-sink-example/
```

## 代码规范

### TypeScript

```
- 严格模式
- ESM (type: "module")
- 显式类型导入/导出
- Zod 验证器用于 API 边界
```

### 数据库

```
- Drizzle ORM 定义表
- 每次修改生成迁移
- 迁移按序号递增
- 包含 JSON 快照
```

```
- Vitest 单元测试 (server/src/__tests__/)
- Playwright E2E (tests/e2e/)

```

```
- 分支命名: feature/xxx, fix/xxx
- 提交消息: 语义化
- PR 需要 Greptile 审查
- 需要声明是否使用 AI 辅助
```

## 深度使用建议

### 1. 使用 Paperclip 管理你的 AI 虚拟公司

```
推荐流程:

第一步: 创建公司
  → 使用 CLI 或 UI 创建公司
  → 配置 Issue 前缀
  → 设置月度预算

第二步: 搭建组织架构
  → 招聘 CEO Agent (Claude Code / Codex)
  → CEO 招聘下属 Agent
  → 设置 reportsTo 关系
  → 配置每个 Agent 的角色、能力、适配器

第三步: 设置目标和项目
  → 创建公司级 Goal
  → 创建 Projects 对齐到 Goal
  → 创建团队级和 Agent 级 Goal

第四步: 建立工作流
  → 设置 Board 审批策略
  → 配置 Issue 执行策略
  → 设置预算控制

第五步: 自动化
  → 创建 Routine (定时扫描、自动报告等)
  → 配置 Webhook 触发器
  → 设置 Routine 变量

第六步: 监控和治理
  → 通过 Dashboard 监控状态
  → 通过 Inbox 处理通知
  → 通过 Costs 页面追踪成本
  → 通过 Activity Log 审查操作
```

### 2. 利用 MCP 进行 Agent 自治

```
Agent 通过 MCP 可以:
  - 自主创建和领取任务
  - 管理自己的评论和文档
  - 请求审批
  - 查询其他 Agent 和项目状态

配置:
  npx paperclip-mcp-server --api-key pcp_xxx
```

### 3. 利用插件扩展能力

```
推荐创建的插件:
  1. 知识库插件 - 连接内部文档
  2. 监控插件 - 连接外部监控系统
  3. 通知插件 - 连接 Slack/邮件
  4. 数据分析插件 - 自定义报表
  5. 审计插件 - 合规审计日志
```
