# Paperclip 项目概览

## 一句话定义

Paperclip 是一个**开源的 AI 智能体公司控制平面**（Control Plane），用于编排和管理由 AI Agent 组成的自治公司。它不是 Agent 运行时，不是 SaaS 平台，而是管理和治理 AI Agent 团队的基础设施。

## 核心能力

| 能力 | 说明 |
|------|------|
| **组织架构管理** | 通过 `reportsTo` 字段构建层级化的 Agent 组织树，支持 CEO/CTO/工程师等 11 种角色 |
| **任务(Issue)系统** | 完整的任务生命周期：backlog → todo → in_progress → in_review → done，支持子任务、阻塞关系、标签 |
| **心跳(Heartbeat)调度** | 定时/按需/指派触发 Agent 执行，带会话管理、工作区隔离、日志收集 |
| **目标(Goal)层级** | 公司 → 团队 → Agent → 任务四级目标对齐体系 |
| **项目(Project)管理** | 项目看板、工作区（git worktree）、环境变量管理 |
| **例行任务(Routine)** | Cron 定时 / Webhook 触发的自动化任务，带并发策略和补执行策略 |
| **预算控制** | 公司/Agent/项目三级别预算，软警告+硬停止，月度/生命周期两种窗口 |
| **审批工作流** | 招聘 Agent、CEO 策略审批、预算覆盖等关键操作需 Board 审批 |
| **插件系统** | 完整的插件 SDK，支持数据读写、事件订阅、定时任务、工具注册、UI 扩展 |
| **适配器系统** | 可扩展的 Agent 运行时适配层，已支持 Claude/Codex/Gemini/Cursor/OpenClaw 等 |
| **公司可移植性** | 完整的导入/导出功能，含数据清洗和版本迁移 |
| **实时通信** | WebSocket 推送运行状态、Agent 状态变更，UI 实时刷新 |

## 技术栈概览

| 层次 | 技术 |
|------|------|
| **后端** | Node.js 20+, Express 5, TypeScript 5.7 |
| **数据库** | PostgreSQL（开发用 PGlite 嵌入式）, Drizzle ORM |
| **前端** | React 19, Vite 6, TanStack Query, Tailwind CSS 4, Radix UI |
| **认证** | Better Auth（支持本地受信模式和认证模式） |
| **包管理** | pnpm 9.15 (monorepo workspace) |
| **测试** | Vitest (单元), Playwright (E2E), Promptfoo (AI评估) |
| **通信** | REST API + WebSocket + MCP (Model Context Protocol) |
| **部署** | Docker / 本地 / Tailscale |

## 运行模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| `local_trusted` | 默认模式，无需登录 | 本地开发、个人使用 |
| `authenticated` | 需要 Better Auth 认证 | 团队部署、公网暴露 |

## 数据隔离策略

所有核心实体都以 `companyId` 为作用域，实现公司级别的数据隔离。插件是全局的（跨公司共享），但插件状态按 scope 隔离。

## 关键设计原则

1. **任务即通信**：Agent 之间只通过 Issue + Comment 交流，没有即时聊天系统
2. **层级追溯**：所有工作通过 Issue → Project → Goal → Company 向上追溯
3. **Board 治理**：人类(Board)拥有无限制访问权限，可以暂停/恢复 Agent、覆盖决策、设定预算
4. **显式错误**：系统暴露问题而非静默修复，崩溃恢复需要手动操作
5. **插件扩展**：通过插件系统扩展知识库、适配器、收入追踪等功能
6. **原子任务分配**：通过 `checkoutRunId` 实现任务原子分配，防止并发冲突
