# Paperclip 数据模型深度分析

## 数据库概览

- **ORM**: Drizzle ORM
- **数据库**: PostgreSQL (开发用 PGlite)
- **迁移数量**: 53 个 (0000-0052)
- **表数量**: 70+
- **Schema 目录**: `packages/db/src/schema/`

## 核心实体关系图

## 核心表结构详解

### Company (公司)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid PK | |
| name | text NOT NULL | 公司名称 |
| description | text | 公司描述 |
| status | text | `active` / `paused` / `archived` |
| pauseReason | text | `manual` / `budget` / `system` |
| issuePrefix | text UNIQUE | Issue 前缀 (如 "PAP") |
| issueCounter | integer | Issue 序号计数器 |
| budgetMonthlyCents | integer | 月度预算（美分） |
| spentMonthlyCents | integer | 已花费（美分） |
| requireBoardApprovalForNewAgents | boolean | 新Agent是否需要Board审批 |
| feedbackDataSharingEnabled | boolean | 反馈数据共享开关 |
| brandColor | text | 品牌颜色 |

### Agent (智能体)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid PK | |
| companyId | uuid FK | 所属公司 |
| name | text | Agent 名称（公司内唯一） |
| role | text | 角色：ceo/cto/cmo/cfo/engineer/designer/pm/qa/devops/researcher/general |
| title | text | 职位头衔 |
| icon | text | 图标（42种可选） |
| status | text | active/paused/idle/running/error/pending_approval/terminated |
| reportsTo | uuid FK→agents | 直属上级（组织树） |
| capabilities | text | 能力描述 |
| adapterType | text | 适配器类型：process/http/claude_local/codex_local/gemini_local/... |
| adapterConfig | jsonb | 适配器配置（JSON） |
| runtimeConfig | jsonb | 运行时配置（JSON） |
| budgetMonthlyCents | integer | Agent 级月度预算 |
| spentMonthlyCents | integer | Agent 级已花费 |
| permissions | jsonb | 权限配置 |
| lastHeartbeatAt | timestamp | 最后心跳时间 |
| metadata | jsonb | 自定义元数据 |

**关键业务规则**：
- `reportsTo` 形成组织树，系统会检测循环引用
- 同公司内 Agent 名称必须唯一（URL规范化后比较）
- 配置变更有版本追踪（AgentConfigRevision）
- API Key 使用 `pcp_` 前缀，SHA-256 哈希存储
- `spentMonthlyCents` 从 CostEvent 动态计算

### Issue (任务/工单)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | uuid PK | |
| companyId | uuid FK | 所属公司 |
| projectId | uuid FK | 所属项目 |
| goalId | uuid FK | 关联目标 |
| parentId | uuid FK→issues | 父任务（子任务树） |
| title | text | 标题 |
| description | text | 描述 |
| status | text | backlog/todo/in_progress/in_review/done/blocked/cancelled |
| priority | text | critical/high/medium/low |
| assigneeAgentId | uuid FK→agents | 指派的Agent |
| issueNumber | integer | 公司内序号 |
| identifier | text UNIQUE | 人类可读ID（如 PAP-42） |
| checkoutRunId | uuid FK→heartbeat_runs | **原子分配锁** |
| executionRunId | uuid FK→heartbeat_runs | 当前执行的运行 |
| executionWorkspaceId | uuid FK→execution_workspaces | 执行工作区 |
| originKind | text | manual / routine_execution |
| requestDepth | integer | 跨团队委派跳数 |
| billingCode | text | 成本归属码 |
| executionPolicy | jsonb | 执行策略（normal/auto + stages） |
| executionState | jsonb | 执行状态（idle/pending/changes_requested/completed） |

**索引设计**（12 个索引）：
- `(companyId, status)` - 按状态查询
- `(companyId, assigneeAgentId)` - 按指派查询
- `(parentId)` - 子任务树
- `(projectId)` - 按项目筛选
- `(originKind, originId)` - 来源追踪
- `(executionWorkspaceId)` - 工作区关联
- `(identifier) UNIQUE` - 唯一标识符
- `GIN trigram` - 标题/描述全文搜索

## 数据库迁移策略

- 迁移文件位于 `packages/db/drizzle/` 目录
- 53 个迁移文件 (0000-0052)，每个附带 JSON 快照
- 使用 Drizzle Kit 自动生成迁移
- 开发模式自动运行迁移
- 支持备份/恢复 (`scripts/backup-db.sh`)

## 二次开发要点

1. **新增表**：在 `packages/db/src/schema/` 中定义，然后运行 `pnpm db:generate` 生成迁移
2. **修改表**：直接修改 schema 文件，重新生成迁移
3. **关联关系**：注意 `companyId` 的数据隔离
4. **枚举扩展**：在 `packages/shared/src/constants.ts` 中添加
5. **类型定义**：在 `packages/shared/src/types/` 中同步更新
6. **验证器**：在 `packages/shared/src/validators/` 中添加 Zod 验证
