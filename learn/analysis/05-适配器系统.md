# Paperclip 适配器系统深度分析

## 适配器架构概览

适配器是 Paperclip 与不同 Agent 运行时之间的桥梁。Paperclip 本身不运行 Agent，而是通过适配器连接到各种执行环境，Agent 通过适配器 "打电话回家" 汇报状态。

```
Paperclip 控制平面
    │
    ├── 适配器注册表 (可变注册表，支持动态加载)
    │     │
    │     ├── 内置适配器 (10个)
    │     └── 外部适配器 (插件提供的 ui-parser.js)
    │
    ├── 服务端适配器 (server/src/adapters/)
    │     ├── 注册表
    │     ├── 插件加载器
    │     ├── 进程执行器
    │     └── HTTP 执行器
    │
    └── UI 适配器 (ui/src/adapters/)
          ├── 注册表
          ├── 动态加载器
          └── 标准输出解析器
```

## 适配器接口详解

### 服务端适配器

```typescript
// 核心接口
interface ServerAdapter {
  // 执行 Agent
  execute(params: ExecuteParams): Promise<ExecuteResult>

  // 测试环境
  testEnvironment(config: AdapterConfig): Promise<TestResult>

  // 检测模型
  detectModel(config: AdapterConfig): Promise<ModelInfo>

  // 技能同步
  syncSkills(agentId: string, skills: Skill[]): Promise<void>
}

interface ExecuteParams {
  agent: Agent
  context: ContextSnapshot
  adapterConfig: AdapterConfig
  workspace: ExecutionWorkspace
  taskKey: string
  sessionId?: string  // 会话恢复
}

interface ExecuteResult {
  exitCode: number
  stdout: string
  stderr: string
  usage?: TokenUsage
  resultJson?: any
}
```

### UI 适配器

```typescript
interface UIAdapter {
  // 适配器类型标识
  type: string

  // 显示名称
  displayName: string

  // 配置表单字段
  configFields: ConfigField[]

  // 标准输出解析器 (解析 Agent 运行日志为结构化数据)
  parseStdout(stdout: string): TranscriptEntry[]

  // 图标
  icon?: string
}
```

### CLI 适配器

```typescript
interface CLIAdapter {
  // 心跳运行输出格式化
  formatHeartbeatRun(run: HeartbeatRun): string
}
```

### UI 注册表

```typescript
// ui/src/adapters/registry.ts
// 内置 10 个适配器在模块加载时注册

const adaptersByType = new Map<string, UIAdapter>()

registerUIAdapter(adapter: UIAdapter): void
unregisterUIAdapter(type: string): void
findUIAdapter(type: string): UIAdapter | undefined
listUIAdapters(): UIAdapter[]
```

**动态加载机制**：
```
请求适配器 → 不在注册表中？
  → 返回 lazy-loading bridge
  → 首次 stdout 行触发 loadDynamicParser()
  → 加载中回退到通用 process 适配器
  → 加载完成替换为实际适配器
```

**外部覆盖机制**：
```
外部插件提供 ui-parser.js:
  ├── 覆盖内置适配器:
  │   ├── generation counter 防止旧版覆盖
  │   ├── 外部解析器替换内置
  │   └── 停用时恢复原始内置
  │
  └── 新增外部适配器:
      └── 注册为 lazy-loading bridge
```

## 创建自定义适配器

### 步骤

1. **在 `packages/adapters/` 下创建新包**

```
packages/adapters/my-agent-local/
├── package.json         # name: @paperclipai/adapter-my-agent-local
├── tsconfig.json
└── src/
    ├── index.ts
    ├── server/
    │   └── index.ts     # 实现 ServerAdapter 接口
    ├── ui/
    │   ├── index.ts     # 实现 UIAdapter 接口
    │   └── config-fields.tsx
    └── cli/
        └── index.ts     # 实现 CLIAdapter 接口
```

2. **实现服务端适配器**

```typescript
// src/server/index.ts
export const myAgentServerAdapter: ServerAdapter = {
  async execute(params) {
    // 1. 构建命令行
    // 2. 执行进程
    // 3. 收集输出
    // 4. 返回结果
  },

  async testEnvironment(config) {
    // 检查依赖是否安装
  },

  async detectModel(config) {
    // 检测当前使用的模型
  }
}
```

3. **实现 UI 适配器**

```typescript
// src/ui/index.ts

  type: 'my_agent_local',
  displayName: 'My Agent',
  configFields: [
    { key: 'model', label: 'Model', type: 'text' },
    // ...

```

4. **注册到系统**

```typescript

// ui/src/adapters/registry.ts - UI 注册

registerUIAdapter(myAgentUIAdapter)
```

### 参考实现

最简单的参考是 `packages/adapters/claude-local/`，它是最成熟的本地适配器实现。

### 外部适配器 (无需修改源码)

如果不想修改 Paperclip 源码，可以通过插件系统提供外部适配器的 UI 解析器：

```
1. 创建插件，在 manifest 中声明 adapter 支持
2. 提供 ui-parser.js 文件
3. 插件加载时自动注册到 UI 适配器注册表
```

参考文档：`docs/adapters/external-adapters.md`

## 适配器与工作区的交互

```
Agent 执行流程中的适配器调用链:

1. 心跳服务 → 选择适配器 (agent.adapterType)
2. 解析工作区 → 适配器获取工作目录
3. 构建 Agent 指令 → 适配器获取指令包
4. 技能注入 → 适配器同步技能到 Agent
5. 执行 → 适配器启动 Agent 进程
6. 监控 → 适配器报告状态/日志
7. 完成 → 适配器返回结果/使用量
```

6. **计费集成**：通过 adapter-utils 的 billing 扩展
