## 后端架构详解

### 1. 启动流程 (`server/src/index.ts`)

```
1. 加载配置 → 解析环境变量和运行时配置
2. 启动嵌入式 PGlite → 开发模式自动创建 PostgreSQL 实例
3. 初始化数据库连接 → Drizzle ORM 客户端
4. 运行数据库迁移 → 53 个迁移文件按序执行
5. 创建 Express App → 挂载中间件和路由
6. 启动 HTTP 服务器 → 默认端口 3100
7. 注册 WebSocket 升级处理 → 实时事件推送
8. 加载插件系统 → 发现并激活已安装插件
9. 启动定时任务调度器 → Routine Cron 执行
```

### 2. Express 中间件栈 (`server/src/app.ts`)

```typescript
// 中间件执行顺序（关键）
1. JSON Body Parser       // 10MB 限制（导入/导出）
2. HTTP Logger            // 请求日志
3. Private Hostname Guard // 认证部署的私有主机名保护
4. Actor Middleware       // 解析认证上下文
5. BetterAuth Session     // 会话处理（认证模式）
6. Board Mutation Guard   // API 路由的 Board 操作守卫
```

### 4. 服务层 (80+ 服务文件)

核心服务按功能域分组：

#### Agent 管理域
- `agents.ts` (24k) - Agent CRUD、配置管理
- `agent-instructions.ts` (26k) - 指令包管理（managed/external 两种模式）
- `heartbeat.ts` (159k) - **最大的服务文件**，心跳执行引擎

#### 任务管理域
- `issues.ts` (89k) - Issue 生命周期、原子分配
- `execution-workspaces.ts` (26k) - 执行工作区管理
- `execution-workspace-policy.ts` (8.9k) - 工作区访问策略
- `issue-execution-policy.ts` (12k) - Issue 执行策略（签收阶段）

#### 自动化域
- `routines.ts` (57k) - 例行任务调度、触发器管理
- `cron.ts` (11k) - Cron 表达式解析和调度

#### 财务域
- `budgets.ts` (32k) - 预算策略执行
- `costs.ts` (17k) - 成本追踪和报告

#### 治理域
- `approvals.ts` (9.6k) - 审批工作流
- `documents.ts` (18k) - 文档管理

#### 插件域（8 个服务文件）
- `plugin-loader.ts` (70k) - 插件发现和加载
- `plugin-host-services.ts` (42k) - 宿主服务暴露
- `plugin-worker-manager.ts` (41k) - Worker 进程管理
- `plugin-lifecycle.ts` (29k) - 生命周期状态机
- `plugin-job-scheduler.ts` (22k) - 作业调度
- `plugin-event-bus.ts` (15k) - 事件分发
- `plugin-tool-registry.ts` (15k) - 工具注册
- `plugin-tool-dispatcher.ts` (15k) - 工具调度

### 3. 状态管理策略

| 机制 | 用途 |
|------|------|
| TanStack Query | 服务端状态管理（所有 API 数据） |
| React Context | 全局状态（公司选择、主题、对话框、Toast） |
| localStorage | 持久化选择（选中的公司 ID） |
| WebSocket | 实时数据推送，智能失效缓存 |

### 4. 实时更新系统 (LiveUpdatesProvider)

```
WebSocket 连接 → /api/companies/{id}/events/ws
     │
     ├── 事件类型:
     │   ├── heartbeat.run.log       # 运行日志
     │   ├── heartbeat.run.queued    # 运行入队
     │   ├── heartbeat.run.status    # 运行状态变更
     │   ├── heartbeat.run.event     # 运行事件
     │   ├── agent.status            # Agent 状态变更
     │   └── activity.logged         # 活动日志
     │
     ├── 智能缓存失效:
     │   └── 根据事件类型选择性失效 TanStack Query 缓存
     │
     ├── Toast 通知:
     │   ├── Issue 创建/更新/评论
     │   ├── Agent 状态变更（错误）
     │   └── 运行完成/失败/超时
     │
     └── 防洪策略:
         ├── 当前页面事件不弹 Toast
         ├── 冷却期（10秒内每类最多3个）
         └── 重连后2秒静默期
```

## MCP 服务器架构

```
paperclip-mcp-server (bin)
  │
  ├── 传输层: stdio
  │
  └── 工具集:
      ├── 身份: paperclipMe, paperclipInboxLite
      ├── Agent: paperclipListAgents, paperclipGetAgent
      ├── Issue: CRUD + checkout/release
      ├── Comment: CRUD + 增量加载
      ├── Document: CRUD + 版本恢复
      ├── Project: list/get
      ├── Goal: list/get
      ├── Approval: 完整审批流程
      └── 逃生舱: paperclipApiRequest (通用 API 调用)
```

MCP 服务器让外部 AI Agent（如 Claude Code）可以直接通过标准 MCP 协议与 Paperclip 交互，实现 Agent 自主管理任务。

## 关键架构模式总结

| 模式 | 应用位置 | 说明 |
|------|----------|------|
| **Monorepo Workspace** | 全项目 | pnpm workspace 管理所有子包 |
| **Adapter Registry** | Server + UI | 可变注册表，支持动态加载外部适配器 |
| **Plugin Worker** | Server | 每个插件独立 Worker 进程，JSON-RPC 通信 |
| **Event Bus** | Plugin 系统 | 发布/订阅模式，29 种领域事件 |
| **Atomic Checkout** | Issue 系统 | `checkoutRunId` 实现无锁并发控制 |
| **Session Continuity** | Heartbeat | Agent 会话持久化，跨心跳保持上下文 |
| **Goal Hierarchy** | 全系统 | 公司→团队→Agent→任务四级目标对齐 |
| **Scope-based Isolation** | 预算/插件/权限 | 多级别作用域隔离 |
| **Smart Cache Invalidation** | 前端 | WebSocket 驱动的 TanStack Query 缓存失效 |
| **Generation Counter** | Adapter 加载 | 防止旧版异步加载覆盖当前状态 |
