# Paperclip AI 虚拟公司方向探索报告

## 核心判断

Paperclip 的本质是一个 **任务驱动的层级组织编排引擎**。它的核心抽象是：

```
Board（人类）→ 公司 → 组织树（Agent）→ 项目 → 目标 → 任务 → 子任务
```

任何可以映射到这个抽象的工作模式，都可以用 Paperclip 来管理。关键约束是：

| 能做 | 不能做 |
|------|--------|
| 多层级任务分解和委派 | 实时双向对话/即时通讯 |
| 定时/事件触发的自动化 | 直接管理代码仓库/文件系统 |
| Agent 之间的任务协调 | 视频处理/音频流处理 |
| 预算管控和成本归因 | 用户界面交互（纯 API/CLI 驱动） |
| 审批和治理 | 外部金融交易执行 |
| 可复现的组织模板 | 持续运行的后台服务 |

## 公司方向分类

按 Paperclip 原生能力和适配器生态的匹配度，将公司方向分为三个梯队：

### 第一梯队：原生适配，开箱即用

这些方向直接匹配 Paperclip 的核心能力：层级任务分解、Agent 委派、代码/文本产出。

---

### 1. 软件开发公司

**最适合的方向。Paperclip 的默认模板就是为此设计的。**

```
Board（你）
  └── CEO（战略规划、任务分解）
        ├── CTO（技术架构、代码审查）
        │     ├── 高级工程师 ×2（核心功能开发）
        │     ├── 工程师 ×2（功能实现、Bug 修复）
        │     └── DevOps（CI/CD、基础设施）
        ├── PM（需求管理、优先级排序）
        │     └── QA（测试、质量保证）
        ├── UX 设计师（界面设计、设计系统）
        └── 研究员（技术调研、方案评估）
```

**适配器配置**：
| Agent | 适配器 | 模型 |
|-------|--------|------|
| CEO | claude_local | Claude Opus/Sonnet |
| CTO | claude_local | Claude Sonnet |
| 工程师 | claude_local / codex_local | Claude Sonnet / Codex |
| DevOps | claude_local | Claude Sonnet |
| QA | claude_local | Claude Sonnet |
| PM | claude_local | Claude Sonnet |
| 设计师 | gemini_local | Gemini Pro |
| 研究员 | claude_local | Claude Sonnet |

**Routine 自动化**：
```
- 每日站会总结：Cron 9:00 UTC → CEO 汇报进度
- 代码审查队列：Webhook → CTO 审查 PR
- 依赖安全扫描：Cron 每周一 → DevOps 扫描
- 测试覆盖率报告：Cron 每日 → QA 生成报告
- 技术债务追踪：Cron 每月 → 研究员评估
```

**预算策略**：
```
公司月度预算：$500
  ├── CTO 团队：$300（工程师贵，需高预算）
  ├── PM 团队：$50
  ├── 设计师：$80
  └── 研究员：$70
```

**执行工作区**：git worktree 隔离，每个 Issue 独立分支。

**关键指标**：Issue 完成率、代码审查周期、Bug 修复时间、成本/Issue。

### 2. 内容创作工厂

**高频产出、流水线作业。完美匹配 Paperclip 的任务委派模型。**

```
Board
  └── CEO（选题规划、质量把控）
        ├── 主编（内容审核、编辑润色）
        │     ├── 撰稿人 ×3（文章撰写）
        │     └── 翻译（多语言内容）
        ├── SEO 策略师（关键词、优化）
        ├── 社媒运营（分发策略）
        └── 研究员（素材收集、事实核查）
```

**工作流**：
```
CEO 制定选题 → 创建 Issue（标题+要求）
  → 主编分解为子任务
    → 撰稿人领取并撰写
    → 研究员提供素材/数据
  → 主编审核
  → SEO 优化
  → 翻译多语言版本
  → 社媒运营制定分发计划
```

**Routine 自动化**：
```
- 每日选题会：Cron 8:00 → CEO 审核选题池
- 热点追踪：Cron 每小时 → 研究员抓取热点
- SEO 周报：Cron 每周一 → SEO 策略师报告
- 内容日历：Webhook → 社媒运营排期
- 质量审计：Cron 每月 → 主编评估产出质量
```

**预算策略**：按内容类型设定，长文 $0.5/篇，短文 $0.2/篇。

### 3. 数据分析公司

**分析报告的生产线。从数据采集到可视化到报告撰写。**

```
Board
  └── CEO（项目规划、客户沟通）
        ├── 首席分析师（分析策略、报告审核）
        │     ├── 数据工程师（数据清洗、ETL）
        │     ├── 数据分析师 ×2（建模、分析）
        │     └── 可视化设计师（图表、Dashboard）
        └── 研究员（行业调研、竞品分析）
```

**工作流**：
```
需求 → 数据工程师采集/清洗
     → 数据分析师建模分析
     → 可视化设计师制作图表
     → 首席分析师审核报告
     → 研究员补充行业背景
     → CEO 终审交付
```

**Routine**：
```
- 日报/周报/月报自动生成（Cron）
- 数据源监控异常报警（Webhook）
- 竞品动态追踪（Cron 每日）
```

## 推荐起步路线

### 阶段一：验证核心（第 1-2 周）

选择 **软件开发公司** 或 **内容创作工厂**，原因：
- 原生适配度最高，零二次开发
- 默认模板直接可用
- 能快速验证 Paperclip 的全部核心能力
- 成本可控（$50-100 起步）

**最小启动配置**：
```
3 个 Agent：CEO + 工程师 + QA
月度预算：$50
1 个项目，3-5 个 Issue
```

### 阶段二：扩展验证（第 3-4 周）

- 扩大到 5-8 个 Agent
- 添加 Routine 自动化
- 尝试跨项目协作
- 验证预算控制效果

### 阶段三：多样化尝试（第 5-8 周）

- 搭建第二种类型的公司（如内容创作）
- 对比不同公司类型的运营差异
- 尝试需要插件的方向
- 开始积累公司模板

### 阶段四：深度定制（第 9+ 周）

- 开发自定义插件
- 开发自定义适配器
- 优化特定方向的工作流
- 打包为可复用的公司模板

## 组织架构模式总结

Paperclip 的 `reportsTo` 自引用结构支持四种经典组织模式：

### 1. 扁平模式（2-4 Agent）
```
Board → CEO → [Agent1, Agent2, Agent3]
```
适合：小型项目、个人工具、快速验证

### 2. 层级模式（5-15 Agent）
```
Board → CEO → [CTO → [Eng1, Eng2], CMO → [Writer1], PM → [QA]]
```
适合：中等规模公司、部门化运营

### 3. 矩阵模式（10-30 Agent）
```
Board → CEO → [CTO, CMO, CPO]
CTO 的工程师可以被 CPO 的项目临时指派
通过 Issue 的 assigneeAgentId 实现跨团队协作
```
适合：大型组织、多项目并行

### 4. 自治模式（3-10 Agent）
```
Board → CEO
所有 Agent reportsTo CEO
Agent 自主领取 Issue（checkout 机制）
```
适合：简单任务池、去中心化工作
