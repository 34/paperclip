## 插件发现与安装

### 发现来源

| 来源 | 说明 |
|------|------|
| `local-filesystem` | 扫描 `~/.paperclip/plugins/` 目录 |
| `npm` | 发现 `paperclip-plugin-*` 命名规范的包 |
| `registry` | 预留：远程注册中心 |

### 插件生命周期

```
installed → ready → disabled
              │         │
              ▼         ▼
             error ←── upgrade_pending
              │
              ▼
          uninstalled
```

### 安装流程

```
1. 发现插件 → 解析 manifest
2. 安装 NPM 包 / 加载本地路径
3. 解析入口文件
4. 创建 Worker 进程
5. 能力授权 (基于 manifest 中声明的 capabilities)
6. 同步作业声明
7. 注册事件订阅
8. 发现工具
9. 状态变为 ready
```

## 插件 Manifest

```typescript
interface PaperclipPluginManifestV1 {
  // 唯一标识 (如 "com.example.my-plugin")
  id: string

  // 显示名称
  name: string

  // 版本 (semver)
  version: string

  // API 版本 (当前为 1)
  apiVersion: 1

  // 描述
  description: string

  // 入口文件
  entrypoint: string

  // 类别
  categories: PluginCategory[]  // connector | workspace | automation | ui

  // 声明的能力
  capabilities: PluginCapability[]

  // UI 扩展
  ui?: {
    slots: UISlotDeclaration[]
    assets?: string  // 静态资源目录
  }

  // 事件订阅
  events?: {
    subscribe: EventType[]
  }

  // 定时作业
  jobs?: JobDeclaration[]

  // 工具声明
  tools?: ToolDeclaration[]
}
```

### 工具调度流程

```
Agent 请求调用工具
  → 心跳服务 → ToolDispatcher
  → 查找工具 → 找到 → 路由到插件 Worker
  → JSON-RPC 调用 → 插件执行
  → 返回结果 → Agent 获得响应
```

## JSON-RPC 2.0 协议

### 通信方式

```
Host ←──stdio──→ Worker

标准 JSON-RPC 2.0 消息:
  Request:  { jsonrpc: "2.0", id: "1", method: "...", params: {...} }
  Response: { jsonrpc: "2.0", id: "1", result: {...} }
  Error:    { jsonrpc: "2.0", id: "1", error: { code: -32000, message: "..." } }
  Notify:   { jsonrpc: "2.0", method: "...", params: {...} }
```

### 错误码

| 代码 | 名称 | 说明 |
|------|------|------|
| -32700 | PARSE_ERROR | 解析错误 |
| -32600 | INVALID_REQUEST | 无效请求 |
| -32601 | METHOD_NOT_FOUND | 方法未找到 |
| -32602 | INVALID_PARAMS | 无效参数 |
| -32603 | INTERNAL_ERROR | 内部错误 |
| -32000 | WORKER_UNAVAILABLE | Worker 不可用 |
| -32001 | CAPABILITY_DENIED | 能力被拒绝 |
| -32002 | WORKER_ERROR | Worker 错误 |
| -32003 | TIMEOUT | 超时 |
| -32004 | METHOD_NOT_IMPLEMENTED | 方法未实现 |
| -32099 | UNKNOWN | 未知错误 |

## 二次开发要点

1. **扩展数据源**：创建 connector 类插件，连接外部系统（如 GitHub, Jira, Slack）
2. **扩展工作区**：创建 workspace 类插件，支持新的开发环境
3. **自动化流程**：创建 automation 类插件，实现自定义业务流程
4. **扩展 UI**：创建 ui 类插件，添加自定义页面、小部件、操作按钮
5. **注册 Agent 工具**：让 Agent 可以调用插件提供的工具
6. **事件驱动**：订阅领域事件，实现自动化响应
7. **状态持久化**：利用 scope 分区的状态存储
8. **自定义实体**：存储外部系统的数据关联
