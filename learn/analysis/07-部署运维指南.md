# Paperclip 部署与运维指南

## 部署方式

### 1. 本地开发模式（推荐入门）

```bash
# 前提: Node.js 20+, pnpm 9.15+

# 克隆仓库
git clone https://github.com/paperclipai/paperclip.git
cd paperclip

# 安装依赖
pnpm install

# 运行数据库迁移
pnpm db:migrate

# 启动开发服务器（API + UI 热重载）
pnpm dev
# 默认访问: http://localhost:3100
```

**开发模式特性**：
- 内置 PGlite（嵌入式 PostgreSQL，无需安装）
- Vite 开发服务器（UI 热重载）
- 自动数据库迁移
- 本地受信模式（无需登录）
- 插件热重载

### 2. Docker 部署

#### 快速启动

```bash
docker compose -f docker/docker-compose.quickstart.yml up
```

#### 标准部署

```yaml
# docker/docker-compose.yml
services:
  paperclip:
    build: .
    ports:
      - "3100:3100"
    environment:
      - DATABASE_URL=postgresql://...
      - PAPERCLIP_RUNTIME_MODE=authenticated
    volumes:
      - paperclip-data:/root/.paperclip
```

#### 非信任审查模式

```bash
docker compose -f docker/docker-compose.untrusted-review.yml up
```

### 3. Tailscale 部署

```
适用于通过 Tailscale 网络安全暴露 Paperclip:
```

### 4. 生产部署检查清单

```

```

## 环境变量

### 核心配置

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `PORT` | 服务端口 | `3100` |
| `DATABASE_URL` | PostgreSQL 连接串 | PGlite 自动 |
| `PAPERCLIP_RUNTIME_MODE` | 运行模式 | `local_trusted` |
| `PAPERCLIP_DATA_DIR` | 数据目录 | `~/.paperclip` |
| `NODE_ENV` | 环境 | `development` |

### 认证配置

| 变量 | 说明 |
|------|------|
| `BETTER_AUTH_SECRET` | Better Auth 密钥 |
| `BETTER_AUTH_URL` | Better Auth URL |

### 存储配置

| 变量 | 说明 |
|------|------|
| `STORAGE_PROVIDER` | 存储提供者：`local` / `s3` |
| `S3_BUCKET` | S3 桶名 |
| `S3_REGION` | S3 区域 |
| `S3_ACCESS_KEY` | S3 访问密钥 |
| `S3_SECRET_KEY` | S3 秘密密钥 |

### 秘钥管理

| 变量 | 说明 |
|------|------|
| `SECRETS_PROVIDER` | 秘钥提供者 |
| `SECRETS_ENCRYPTION_KEY` | 加密密钥 |

## 数据库管理

### 迁移

```bash
# 生成新迁移（修改 schema 后）
pnpm db:generate

# 运行迁移
pnpm db:migrate

# 备份数据库
pnpm db:backup
```

### 迁移策略

```
packages/db/drizzle/
├── 0000_initial_migration.sql
├── 0000_snapshot.json
├── 0001_*.sql
├── 0001_snapshot.json
├── ...
└── 0052_*.sql
    └── 0052_snapshot.json

每个迁移:
  - SQL 迁移脚本
  - JSON 快照（用于回滚计算）
  - 按序号递增
```

## 备份与恢复

### 备份

```bash
# 使用内置脚本
pnpm db:backup
# 或
./scripts/backup-db.sh
```

### 公司级别导出/导入

```
导出:
  UI → 公司设置 → 导出
  → 生成 ZIP 包（含所有数据，已清洗敏感信息）

导入:
  UI → 公司 → 导入
  → 上传 ZIP 包 → 预览 → 确认导入
  → 自动迁移数据版本
```

## 安全注意事项

### Agent API Key 管理

```
格式: pcp_{随机字符串}
存储: SHA-256 哈希
权限: 绑定到特定 Agent
用途: Agent 认证、WebSocket 连接
```

### Board 访问控制

```
local_trusted 模式:
  → 所有请求自动为 Board 角色
  → 无需认证

authenticated 模式:
  → Better Auth 会话认证
  → instance_admin 角色
  → 公司成员关系检查
```

### 秘钥安全

```
公司秘钥:
  - 加密 at rest
  - 按需解密
  - Webhook 签名秘钥自动轮换

部署选项:
  - 本地加密 (默认)
  - 外部秘钥管理 (推荐生产环境)
```

## 常见运维操作

### 暂停/恢复公司

```
暂停: PUT /api/companies/{id} → { status: "paused", pauseReason: "manual" }
恢复: PUT /api/companies/{id} → { status: "active" }
注意: 预算暂停只恢复 pauseReason === "budget" 的
```

### 清理工作区

```
执行工作区生命周期:
  active → cleanupEligibleAt → closed
  系统自动标记可清理的工作区
```

## 二次开发部署建议

### 你的 AI 虚拟公司场景

1. **起步**: 使用本地开发模式 + PGlite
2. **小规模**: Docker 部署 + 本地存储
3. **生产级**: Docker + 外部 PostgreSQL + S3 + 认证模式
4. **扩展**: 添加自定义插件和适配器

### 推荐配置

```
开发环境:
  PAPERCLIP_RUNTIME_MODE=local_trusted
  STORAGE_PROVIDER=local
  SECRETS_PROVIDER=local

生产环境:
  PAPERCLIP_RUNTIME_MODE=authenticated
  STORAGE_PROVIDER=s3
  SECRETS_PROVIDER=external
  BETTER_AUTH_SECRET=<strong-secret>
```
