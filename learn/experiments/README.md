# 公司搭建实验

不同方向、不同架构的 AI 虚拟公司搭建尝试。记录设计思路、搭建过程、运行效果和经验教训。

## 实验维度

### 按公司类型
- 软件开发公司（工程师团队、代码审查、CI/CD）
- 内容创作公司（写手、编辑、发布流水线）
- 数据分析公司（分析师、报告生成、可视化）
- 客服运营公司（客服 Agent、工单处理、升级流程）
- 研究机构（研究员、文献分析、报告撰写）

### 按组织架构
- 扁平架构（少量 Agent，直接汇报）
- 层级架构（CEO → VP → Manager → Member）
- 矩阵架构（跨项目协作）
- 去中心化架构（Agent 自主领取任务）

### 按自动化程度
- 手动驱动（Board 全程控制）
- 半自动（Routine 辅助 + 人工审批）
- 全自动（Agent 自治 + 预算约束）

### 按规模
- 微型（2-3 个 Agent）
- 小型（5-10 个 Agent）
- 中型（10-30 个 Agent）
- 大型（30+ 个 Agent）

## 实验文档模板

每个实验建议包含：

```markdown
# 实验名称

## 目标
这次实验要验证什么？

## 设计
- 公司结构：组织架构图
- Agent 配置：角色、适配器、能力
- 项目/目标：任务体系
- 自动化：Routine、Webhook
- 预算：成本控制策略

## 搭建过程
逐步操作记录

## 运行观察
- 运行了多久
- 关键事件
- 成本消耗
- 出现的问题

## 结论
- 成功的方面
- 失败的方面
- 经验教训
- 下一步计划
```

## 文档命名规范

实验项目目录和文档使用中文命名, 格式:

```
实验-序号-简述/

例如:
实验-001-PPT代做自动化服务/
实验-002-软件开发公司扁平架构/
实验-003-内容工厂层级架构/
```

每个实验目录结构参考:

```
实验-XXX-简述/
├── README.md          ← 项目总览 (保留英文, 约定俗成)
├── 项目清单.md        ← 任务跟踪
├── plans/             ← 规划文档
├── research/          ← 调研报告
├── engineering/       ← 工程设计
└── raw/               ← 原始素材
```
