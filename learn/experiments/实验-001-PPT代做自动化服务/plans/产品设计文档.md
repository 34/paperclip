## Problem Statement

中国有大量职场人、学生、创业者需要高质量PPT，但没有时间/能力自己做。现有AI PPT工具（ChatPPT、Gamma、WPS AI等）是自助工具，用户需要自己写提示词、自己调整，效果参差不齐。案例证明客户"自己试过AI发现做得稀巴烂"。市场空缺不在"AI PPT工具"，而在"AI PPT代做服务"——全自动的全包服务。

## Demand Evidence

**间接证据（来自案例）：**
- 某高中辍学退伍军人团队用AI帮人做PPT，年入500万+ RMB
- 每小时14-20条新增订单，做不过来
- 获客成本10元/咨询，随便成交都是100+元
- 团队中50多岁工厂阿姨也能月赚5000（兼职）
- 客单价分布：100-600元常见，商业计划书1000-2000元

**直接证据：暂无。** 这是最大的缺口——还没有验证Paperclip的目标客户是否愿意为自动化服务付钱。

## Status Quo

目标用户当前解决PPT问题的方式：
1. **自己硬做**：用WPS/PowerPoint，花2-5小时，做得痛苦且丑
2. **花钱找人做**：淘宝/闲鱼/小红书上的代做服务商，100-2000元
3. **用AI工具自己做**：Kimi、WPS AI、Gamma等，但效果不稳定

关键洞察：客户付钱给代做服务商，买的不是"PPT文件"，而是"不用操心"。他们要的是把需求丢过去，然后收到一个成品。自助AI工具解决不了这个需求。

## Target User & Narrowest Wedge

**最窄切入点：毕业答辩PPT**（当前4-6月正值高峰期）

理由：
- **当前正是旺季**（4-6月毕业季），不需要等季节窗口
- 结构高度标准化（开题/中期/答辩，框架固定）
- 客单价适中（100-300元，学生决策门槛低）
- 需求明确（学生有论文内容，需要转化为PPT呈现）
- 可量化验证（答辩PPT页数通常15-25页，格式统一）

**后续扩展方向（阶段1验证成功后考虑）：**
- 年终总结/年中汇报PPT（1月/7月高峰）
- 商业计划书/路演PPT（客单价高但复杂度也高）

## Constraints

- 作为Paperclip内部项目，需要2-4周内上线MVP
- 初始投入控制在最小（服务器+API调用成本）
- 团队资源有限（需要复用Paperclip现有AI基础设施）
- 中国市场，需要适配中文场景和本土获客渠道

## Premises

1. **需求形态可能不同** — 客户付钱给"人"和付钱给"机器"是两件事。自动化产品需要先证明客户愿意接受机器交付。B阶段（人工把关）用于验证这一点。
2. **需要最窄切入点** — 不能一开始服务所有类型。从年终总结PPT切入，做到极致再扩展。
3. **核心竞争力重新定义** — 壁垒不在AI生成能力（任何人都能调API），在于需求理解的准确度 + 模板质量 + 交付速度 + 客户信任。
4. **快速MVP验证** — 先用最小成本验证需求，再扩大投入。

## Market Landscape

**AI PPT工具市场已极度拥挤：**
- Gamma：7000万用户，$21亿估值
- ChatPPT：国内"断层第一"
- WPS AI / 百度文库PPT / Kimi PPT / AiPPT / 一点PPT / 麦当秀等
- Tome：融了$8100万，然后**倒闭了**

**关键区分：以上都是自助工具，不是代做服务。** 我们要做的是自动化代做服务，不是又一个PPT生成工具。

**市场规模：** 2025年AI PPT市场约$20亿，预计2033年达$100亿。

## Approaches Considered

### Approach A: 全自动PPT代做平台（终极目标）

完全自动化的PPT代做服务。客户提交需求，系统自动解析、生成大纲、确认、生成PPT、交付。全程不需要人工。

- 规模：M
- 风险：高
- 优点：边际成本接近零，可扩展，真正的差异化
- 缺点：需求理解的准确性是最大挑战，需要大量prompt工程

### Approach B: AI辅助+人工把关的PPT服务（MVP起点）

复刻案例团队的商业模式，用AI大幅提效。人工负责客户沟通和质量把关，AI负责生成。让1个人做出10个人的产出。

- 规模：S
- 风险：低
- 优点：快速验证市场，学习客户真实需求，收入立即可见
- 缺点：天花板是人

### Approach C: PPT工具API（卖水给挖金人）

给PPT服务商提供AI生成API，不做C端。

- 规模：M
- 风险：中
- 优点：不需要自己获客
- 缺点：需要找到B端客户，定价权受限

## Recommended Approach

**B→A 渐进路径：**

1. **阶段1（第1-4周）：方案B — AI辅助人工服务（MVP）**
   - 建立4步prompt链（详见下方 Prompt Chain 规范）
   - 用模板填充法生成PPT（预设计模板 + python-pptx内容填充，详见技术风险）
   - 人工通过微信与客户沟通，AI负责生成
   - 小红书/闲鱼获客测试
   - 目标：验证客户愿意付钱，积累50+订单的需求理解数据
   - **交付方式**：微信发送.pptx文件 + PDF预览版（带水印）

2. **阶段2（第5-12周）：内部工具化**
   - 构建内部操作面板（不是客户前端）：订单管理、prompt链编排、质量检查界面
   - 系统自动解析需求（结构化表单辅助，非纯NLU），人工做最终质量检查
   - 自动化比例提升到70%
   - 目标：日处理10-20单，1人可完成全流程
   - **注意**：阶段2投入取决于阶段1验证结果。如果阶段1需求不成立，不进入阶段2。

3. **阶段3（待定）：方案A — 全自动化**
   - 阶段3的设计将在阶段2完成后基于实际数据重新规划
   - 方向：客户自助下单、确认、下载，90%+订单无需人工介入
   - 仅供参考，不做为当前执行计划的一部分

## Prompt Chain 规范（阶段1核心）

```
Step 1: 需求解析
  输入：客户原始聊天文本（微信消息）
  输出：结构化JSON {
    ppt_type: "graduation_defense" | "annual_review" | "business_plan" | ...,
    topic: string,
    page_count: number,
    audience: string,
    tone: "academic" | "business" | "creative",
    content_source: "user_provided" | "ai_generated" | "mixed",
    deadline: date,
    style_preference: string,
    raw_content: string  // 客户提供的原文/论文摘要等
  }

Step 2: 大纲生成
  输入：Step 1的JSON输出
  输出：逐页大纲 [
    {page: 1, title: "封面", bullets: ["论文题目", "姓名/导师/学校"]},
    {page: 2, title: "研究背景", bullets: ["..."]},
    ...
  ]

Step 3: 内容填充
  输入：大纲 + 客户原始内容
  输出：每页详细内容 [
    {page: 1, layout: "cover", title: "...", subtitle: "..."},
    {page: 2, layout: "text_image", title: "...", body: "...", image_hint: "..."},
    ...
  ]

Step 4: PPT渲染
  输入：内容 + 模板ID
  输出：.pptx文件（通过python-pptx填充预设计模板）
```

**人工介入点：** Step 1后人工确认需求理解是否正确，Step 2后客户确认大纲，Step 4后人工做质量检查再交付。

## 修订与异常处理

- **免费修订**：交付后3次以内免费修改，每次修改记录具体需求变更
- **付费修订**：超过3次按20元/次收费（跟客户提前说明）
- **质量不合格**：客户不满意可全额退款，退款订单用于优化prompt链
- **需求不清**：人工介入澄清，不强制AI猜测
- **交付时效**：标准订单4小时内交付，加急（+50%费用）2小时内交付

### 推荐技术方案：三层架构

**第一层（阶段1 MVP）：ChatPPT/AiPPT API + 人工微调**
- 直接调用ChatPPT或AiPPT的API生成PPT
- 人工做最终质量检查和微调
- 优点：最快上线（1周内可接单），质量有保障
- 缺点：依赖第三方API，成本不可控

**第二层（阶段2 核心建设）：PPTAgent + python-pptx模板填充**
- 部署PPTAgent开源方案（中科院，EMNLP 2025论文）
- 投资设计5-10套高质量.pptx模板（涵盖答辩、年终总结等场景）
- PPTAgent分析参考模板→AI规划内容布局→python-pptx填充
- 优点：完全自主可控，质量达设计师级别（8-9/10），成本只有LLM API费
- 缺点：需要投入模板设计和PPTAgent部署

**第三层（阶段3 自动化极致）：自建模板引擎 + 持续学习**
- 基于阶段1-2积累的订单数据，训练需求理解模型
- 自建模板引擎，支持动态布局变化
- 从客户修改记录中学习偏好，持续优化输出质量

### 关键发现

1. **HTML→PPTX是死路**：HTML/CSS和PowerPoint是完全不同的渲染模型。没有任何转换器能保留足够的设计质量用于专业输出。案例团队的方法（AI生成HTML→浏览器打开→手动复制）本质上还是人肉转换，不可自动化。

2. **模板是质量的唯一决定因素**：python-pptx本身没有任何设计智能。但配合专业设计的模板做"填空"，输出可以达到设计师级别（8-9/10）。关键是要投资在模板设计上。

3. **PPTAgent是最佳开源方案**：中科院ICT-CAS发表在EMNLP 2025，采用"编辑式生成"——分析精美的参考PPT，提取幻灯片模式，然后用AI智能替换内容。设计评分比原始方法高33%，成功率97.8%。

4. **ChatPPT API是最快上线路径**：必优科技提供企业级SaaS API，支持白标集成，双模型架构（Wernicke负责布局设计 + LLM负责内容）。可以最快速度上线验证需求。

5. **AiPPT是豆包等消费级应用的底层引擎**：提供开放API，按量付费，适合快速集成。
