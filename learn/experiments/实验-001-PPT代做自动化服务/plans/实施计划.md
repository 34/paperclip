# PPT 代做自动化服务 — 实施计划

## Context

基于已完成的全部调研报告和工程设计文档（产品设计文档、工程评审、技术调研、渠道分析），将 P1-P7 项目阶段翻译为可直接执行的具体任务。目标：4 周内验证毕业答辩 PPT 自动化代做服务是否可行。

核心挑战：Paperclip 服务器无 Python 运行时，PPT 生成需要独立 Python 微服务 + Plugin 桥接。

---

## 架构概览

```
客户 (微信/闲鱼)
  ↓
企业微信客服 API / 闲鱼人工转发
  ↓
wecom-plugin (Webhook回调 → Issue创建 → 客服Agent唤醒)
  ↓
客服专员 Agent (需求解析 → 报价 → 大纲确认)
  ↓
PPT设计师 Agent (大纲 → 内容填充 → 调用pptgen-plugin)
  ↓
pptgen-plugin → ctx.http.fetch() → Python PPT Service (:5001)
  ↓                                    ↓ python-pptx + 模板
质检员 Agent ← .pptx文件              .pptx 输出
  ↓
客服专员 → wecom-plugin → 企微发文件给客户
```

### Day 4-5 (4/13-14): P5b Plugin 开发（两个 Plugin 可并行）

**pptgen-plugin**

```bash
# 脚手架
pnpm --filter @paperclipai/create-paperclip-plugin build
node packages/plugins/create-paperclip-plugin/dist/index.js \
  @paperclipai/ppt-generator --output packages/plugins/ppt-generator
```

Manifest 声明:
- **capabilities**: `agent.tools.register`, `http.outbound`, `plugin.state.read/write`, `secrets.read-ref`
- **tools** (4个):
  - `generate_ppt` — 调用 Python 服务生成 .pptx
  - `list_templates` — 列出可用模板
  - `generate_watermark_pdf` — 生成水印预览 PDF
  - `call_chatppt_api` — ChatPPT API 降级方案
- **instanceConfigSchema**: `pptGenServiceUrl` (默认 `http://localhost:5001`)

Worker 核心逻辑: 所有 tool 通过 `ctx.http.fetch()` 调用 Python PPT Service。

**wecom-plugin**

```bash
node packages/plugins/create-paperclip-plugin/dist/index.js \
  @paperclipai/wecom-customer-service --output packages/plugins/wecom-customer-service
```

Manifest 声明:
- **capabilities**: `webhooks.receive`, `agent.tools.register`, `http.outbound`, `plugin.state.read/write`, `secrets.read-ref`, `issues.read/create/update`, `issue.comments.create/read`
- **webhooks**: `wecom-callback` (企微消息回调)
- **tools** (3个):
  - `send_wecom_message` — 发送文本/图片消息
  - `send_wecom_file` — 上传+发送文件 (.pptx/.pdf)
  - `get_customer_info` — 查询客户映射信息
- **instanceConfigSchema**: `corpId`, `corpSecretRef`, `kfAccountToken`, `kfEncodingAesKey`

Worker 核心逻辑:
- `onWebhook()` → 解析企微 XML → 查找/创建 Issue → 添加评论 → 唤醒客服 Agent
- 客户映射存储在 `ctx.state` (scopeKind: "company", namespace: "customers")

````markdown
---
name: ppt-generation
description: >
  PPT生成完整工作流: 需求读取 → 模板选择 → 大纲生成 → 内容填充 → PPT渲染。
  供PPT设计师Agent使用。当Issue被分配给自己且label包含ppt-order时触发。
---

# PPT Generation Skill

你是PPT设计师，向客服专员汇报。你的核心职责是根据客户需求生成高质量的毕业答辩PPT。

## 触发条件

当以下条件同时满足时开始工作:
- Issue status = `todo` 或 `in_progress`
- Issue labels 包含 `ppt-order`
- Issue assignee = 你自己

## 完整工作流

### Step 1: 读取客户需求

```
GET /api/issues/{issueId}/documents/customer-requirements
```

解析返回的 JSON，提取以下字段:

| 字段 | 说明 | 示例 |
|------|------|------|
| `ppt_type` | PPT类型 | `graduation_defense` |
| `topic` | 论文题目 | "基于深度学习的图像分类研究" |
| `page_count` | 期望页数 | 20 |
| `audience` | 目标学科 | "计算机科学" |
| `tone` | 风格倾向 | `academic` |
| `content_source` | 内容来源 | `user_provided` |
| `deadline` | 截止时间 | "2026-05-20" |
| `style_preference` | 风格偏好 | "蓝色系" |
| `raw_content` | 客户原文 | 论文摘要/大纲/全文 |

如果没有 `customer-requirements` document，读取 Issue 描述和评论获取需求信息，并自行结构化为上述格式写入 document。

### Step 2: 选择模板

调用 `ppt-generator.list_templates` 工具获取可用模板列表。

根据学科和风格选择模板:

| 条件 | 模板 ID | 说明 |
|------|---------|------|
| 理工科 / 无特殊偏好 | `grad-defense-blue` | 学术蓝，最通用 |
| 农林医学 | `grad-defense-green` | 学术绿 |
| 经管社科 | `grad-defense-gray` | 简约灰 |
| 计算机 / AI / 科技类 | `grad-defense-tech` | 科技蓝 |
| 人文艺术 | `grad-defense-art` | 文艺风 |
| 客户指定蓝色系 | `grad-defense-blue` | 优先满足客户偏好 |
| 客户指定绿色系 | `grad-defense-green` | 优先满足客户偏好 |

在 Issue 评论中记录模板选择及理由。

### Step 3: 生成大纲

基于客户需求和选择模板的 layout 选项，生成逐页大纲。

**毕业答辩PPT标准结构** (15-25页):

```
Page 1:  cover        — 封面 (论文题目 + 姓名/导师/学校/日期)
Page 2:  toc          — 目录 (4-6个主要章节)
Page 3:  content      — 研究背景与意义
Page 4:  content      — 国内外研究现状 (文献综述)
Page 5:  content      — 研究目标与内容
Page 6:  content      — 研究方法/技术路线
Page 7:  two-column   — 实验设计/数据来源
Page 8:  chart        — 实验结果展示 (图表)
Page 9:  chart        — 实验结果分析
Page 10: content      — 讨论 (结果的含义和贡献)
Page 11: content      — 结论与展望
Page 12: ending       — 致谢
Page 13: ending       — Q&A
```

根据 `page_count` 和 `raw_content` 的实际内容调整页数和章节分布。

将大纲写入 Issue document:

```
PUT /api/issues/{issueId}/documents/outline
{
  "title": "PPT大纲",
  "format": "markdown",
  "body": "# PPT大纲\n\n## 模板: grad-defense-tech\n\n| 页码 | 布局 | 标题 | 要点 |\n|------|------|------|------|\n| 1 | cover | ... | ... |\n..."
}
```

在 Issue 评论中通知客服专员确认大纲:

```
@客服专员 大纲已生成，请转客户确认。模板选择: grad-defense-tech（科技蓝）。共15页。
```

将 Issue 状态更新为 `blocked`，说明等待客户确认大纲。

### Step 4: 内容填充

收到客户确认后（Issue 上有新评论确认大纲通过），继续工作。

**首先 checkout Issue:**
```
POST /api/issues/{issueId}/checkout
{ "agentId": "{your-agent-id}", "expectedStatuses": ["blocked"] }
```

逐页填充详细内容:
- **封面**: 提取论文题目、作者姓名、导师姓名、学校名称、答辩日期
- **目录**: 根据大纲自动生成
- **正文页**: 每页文字量 100-200 字，学术语言，数据要有来源标注
- **图表页**: 生成图表标题和数据说明（实际图表由模板占位）
- **致谢**: 根据模板生成标准致谢词
- **Q&A**: 生成"谢谢各位老师！"或"感谢聆听，欢迎提问"

**内容质量要求:**
- 语言正式、准确、简洁
- 术语使用一致
- 数据引用规范
- 逻辑清晰、层次分明
- 避免口语化表达

将内容写入 Issue document:

```
PUT /api/issues/{issueId}/documents/content
{
  "title": "PPT内容",
  "format": "json",
  "body": "[{\"layout\":\"cover\",\"title\":\"基于深度学习的图像分类研究\",\"subtitle\":\"张三 | 导师：李四教授 | XX大学计算机学院\",\"notes\":\"\"},{\"layout\":\"toc\",\"title\":\"目录\",\"items\":[\"研究背景\",\"文献综述\",\"研究方法\",\"实验结果\",\"结论与展望\"]},...]"
}
```

### Step 5: 渲染PPT

调用 `ppt-generator.generate_ppt` 工具:

```
参数: {
  "templateId": "grad-defense-tech",
  "content": [上一步的JSON内容],
  "outputPath": "order-{issueIdentifier}.pptx"
}
```

如果生成失败:
1. 检查 content 格式是否符合模板 layout 要求
2. 修正后重试一次
3. 如果仍然失败，降级调用 `ppt-generator.call_chatppt_api`

将输出文件路径记录在 Issue 评论中:
```
PPT生成完成: order-PAP-42.pptx, 共15页。
```

将 Issue 状态更新为 `in_review`，assignee 改为质检员:
```
PATCH /api/issues/{issueId}
{ "status": "in_review", "assigneeAgentId": "{质检员ID}" }
```

## 修订流程

收到 changes_requested（质检或客户反馈）后:

1. Checkout Issue
2. 读取 `quality-report` document 或客户反馈评论，理解修改要求
3. 针对性修改受影响的页面，**不重新生成全部**
````
