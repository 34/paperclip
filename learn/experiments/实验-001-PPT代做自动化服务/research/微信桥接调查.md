# Investigation: 微信桥接方案可行性分析

**Date:** 2026-04-10
**Branch:** learn
**Status:** DONE_WITH_CONCERNS

---

## 核心发现

**Wechaty 不可用于生产环境。** 所有免费 puppet 协议已被微信封杀，付费 puppet 不稳定且处于灰色地带。更严重的是，主流 Hook 方案 (WeChatFerry, Gewechat) 在2025年已被法律迫停。

**推荐方案已变更：企业微信客服 API 是唯一合规、稳定、免费的路径。**

---

## 方案对比

### 1. Wechaty (原方案)

| 维度 | 评估 |
|------|------|
| 维护状态 | **冻结** — 最后npm发布 2022年5月 (v1.20.2)，无实质更新 |
| 免费Puppet | **全部失效** — UOS协议2025年1月被封，wechat4u不可靠 |
| 付费Puppet | PadLocal ~200元/月 (iPad协议)，WorkPro (企业微信)，Paimon |
| 封号风险 | **中-高** — 微信明确禁止非官方自动化 |
| 文件支持 | PadLocal/WorkPro 支持 .pptx/.pdf，免费puppet不可靠 |
| 语言 | TypeScript (主), Python, Go, Java |
| **结论** | **不推荐** — 框架冻结，免费方案全挂，付费方案灰色地带 |

### 2. Hook/注入工具 (WeChatFerry, Gewechat, wxhelper)

| 工具 | 状态 | 封号风险 |
|------|------|---------|
| WeChatFerry | 社区报告已失效 | 高 |
| Gewechat | **README明确写着"因相关法律原因，本项目不再维护及可用"** | 高 |
| wxhelper | 维护但高风险 | 高 |
| pc-wechat-hook | 2025年活跃 | 高 |

**2025年上半年微信封禁约2780万个违规账号，同比增长32%。所有Hook方案都不适合商业用途。**

### 3. wxauto (UI自动化) — 唯一可行的个人微信方案

| 维度 | 评估 |
|------|------|
| 原理 | Windows UIAutomation，模拟点击/键盘，**无注入无Hook** |
| GitHub | cluic/wxauto, ~6,665 stars |
| 最后更新 | 2025年2月 |
| 封号风险 | **低-中** — 模拟真实用户操作，社区称"零封号" |
| 文件支持 | 是 (通过UI自动化发送文件) |
| 平台 | **仅Windows** — 需要专用Windows机器 |
| 限制 | 微信窗口必须保持打开可见，消息类型有限，速度较慢 |
| 成本 | 免费 (Apache 2.0)，有Plus/Pro付费版 |
| **结论** | **MVP可用，但需专用Windows机器，不适合规模化** |

### 4. 企业微信客服 API (推荐方案)

| 维度 | 评估 |
|------|------|
| 原理 | 微信官方API，通过企业微信向个人微信用户发送消息 |
| 消息类型 | 11种：文本、图片、语音、视频、**文件**、链接、小程序卡片等 |
| 文件支持 | **是** — 文件上传/下载API，支持 .pptx/.pdf |
| 客服功能 | 个人微信用户无需加好友即可发起咨询 |
| 封号风险 | **零** — 官方合规 |
| 成本 | **免费** (企业微信基础版) |
| 限制 | 消息显示来自企业账号而非个人；需企业认证(300元/年)；API访问需企业验证 |
| 2025新功能 | 消息推送V3、AI客服API、智能工作流 |
| **结论** | **唯一适合商业运营的合规方案** |

---

## 推荐方案：企业微信客服 API

### 为什么不用个人微信自动化

1. **法律风险**：Hook方案已被迫停，Gewechat明确因法律原因停止
2. **封号风险**：2780万账号被封，商业运营不可承受
3. **稳定性**：Wechaty冻结，免费puppet全挂，付费puppet运营商不稳定
4. **客户信任**：用企业微信发消息比个人号更显专业

### 为什么企业微信客服API是正确选择

1. **合规**：官方API，零封号风险
2. **免费**：基础版不花钱
3. **文件传输**：原生支持发送 .pptx/.pdf
4. **无需好友**：客户扫码即可开始对话
5. **规模化**：无消息频率限制（客服消息）
6. **专业形象**：企业认证比个人号更有说服力
7. **AI集成**：2025年新增AI客服API

### 架构变更

原方案 (wechat-plugin via Wechaty):
```
客户个人微信 ←→ Wechaty ←→ Paperclip Plugin
```

新方案 (企业微信客服 API):
```
客户个人微信 ←→ 企业微信客服 ←→ wecom-plugin ←→ Paperclip Plugin
```

## 实施方案

### Phase 1 (MVP, 第1周)

1. 注册企业微信 (免费)
2. 开通微信客服功能
3. 开发 `wecom-plugin` 替代原 `wechat-plugin`

```typescript
// wecom-plugin manifest
const manifest: PaperclipPluginManifestV1 = {
  id: "wecom-customer-service",
  apiVersion: 1,
  version: "0.1.0",
  displayName: "WeCom Customer Service Bridge",
  description: "Bridge WeCom customer service messages to Paperclip issues",
  categories: ["messaging", "integration"],
  capabilities: ["webhooks", "tools", "state"],
  entrypoints: { worker: "./worker.ts" },

  tools: [
    {
      name: "send_wecom_message",
      displayName: "发送客服消息",
      description: "Send a text message to a WeChat user via WeCom customer service",
      parametersSchema: {
        type: "object",
        properties: {
          externalUserId: { type: "string" },
          msgType: { type: "string", enum: ["text", "image", "file", "link"] },
          content: { type: "string" }
        },
        required: ["externalUserId", "msgType", "content"]
      }
    },
    {
      name: "send_wecom_file",
      displayName: "发送文件",
      description: "Upload and send a file (pptx/pdf) to a WeChat user",
      parametersSchema: {
        type: "object",
        properties: {
          externalUserId: { type: "string" },
          filePath: { type: "string" },
          fileName: { type: "string" }
        },
        required: ["externalUserId", "filePath", "fileName"]
      }
    },
    {
      name: "get_customer_info",
      displayName: "获取客户信息",
      description: "Get WeChat customer profile from WeCom",
      parametersSchema: {
        type: "object",
        properties: {
          externalUserId: { type: "string" }
        },
        required: ["externalUserId"]
      }
    }
  ],

  webhooks: [
    {
      endpointKey: "wecom-callback",
      displayName: "企业微信回调",
      description: "Receives WeCom customer service message callbacks"
    }
  ]
};
```

4. 企业微信客服 API 关键端点:

```
# 接收消息回调 (webhook)
POST /api/plugin/webhooks/{webhookKey}
  ← 企业微信推送客户消息事件

# 发送客服消息
POST https://qyapi.weixin.qq.com/cgi-bin/kf/send_msg_on_event?access_token=TOKEN
  → 发送文本/图片/文件/链接给客户

# 上传临时素材(文件)
POST https://qyapi.weixin.qq.com/cgi-bin/media/upload?access_token=TOKEN&type=file
  → 上传pptx/pdf获取media_id

# 获取客服账号列表
GET https://qyapi.weixin.qq.com/cgi-bin/kf/account/list?access_token=TOKEN
```

### Phase 2 (可选，验证后)

如果客户反馈"不想跟企业号聊天"，可以用 wxauto 作为个人微信的补充渠道:
- 在专用Windows机器上运行wxauto
- 仅用于获客初期（小红书/闲鱼引流的第一个触点）
- 一旦客户进入服务流程，引导到企业微信完成正式沟通和交付

需要更新的内容:

| 原方案 | 新方案 | 变更原因 |
|--------|--------|---------|
| wechat-plugin (Wechaty) | wecom-plugin (企业微信客服API) | Wechaty冻结，Hook方案法律风险 |

## 调查结论

| 维度 | 评估 |
|------|------|
| Wechaty可行性 | **不可行** — 框架冻结，免费puppet全挂 |
| Hook方案可行性 | **不可行** — 法律风险，已有项目被迫停 |
| wxauto可行性 | **MVP可用** — 但需Windows专用机，不适合规模化 |
| 企业微信客服API | **推荐** — 合规、免费、稳定、支持文件传输 |
| 整体风险 | 从"高"降到"低" |

**STATUS: DONE_WITH_CONCERNS**

CONCERNS:
1. 企业微信需要企业资质注册 (个体工商户或公司)
2. 客户可能对企业号有抵触 (需要A/B测试验证)
3. 客服消息有48小时回复窗口限制 (用户最后一次交互后48小时内可主动发送)
